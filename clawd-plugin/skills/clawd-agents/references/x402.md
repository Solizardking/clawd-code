# Solana x402 Reference

Source: `/Users/8bit/clawd-code/clawd-agents/clawd-x402`

The x402 package implements programmable pay-per-call agents on Solana. A caller receives a `402 Payment Required` challenge, signs a Solana token transfer, retries with proof, and the gateway verifies, settles, receipts, and forwards the request.

## Flow

1. Caller requests an agent endpoint.
2. Gateway returns a machine-readable payment challenge.
3. Wallet signs an SPL token or p-token transfer.
4. Gateway verifies blockhash, mint, recipient, amount, payer, and batch shape.
5. Gateway settles and writes a receipt.
6. Gateway strips payment credentials before forwarding to the upstream agent.
7. Caller receives the upstream response plus payment receipt metadata.

## Moving Parts

| Part | Source path | Purpose |
|---|---|---|
| Worker gateway | `worker/src/index.ts` | Routes health, registry reads, facilitator calls, A2A cards, and paid invocations |
| Protocol negotiation | `worker/src/protocols/negotiate.ts` | Chooses x402, MPP, or AP2 from request headers |
| x402 adapter | `worker/src/protocols/x402.ts` | Builds challenges and response receipts |
| Solana verifier | `worker/src/solana/x402.ts` | Verifies signed Solana transactions against challenges |
| Facilitator | `worker/src/solana/facilitator.ts` | Provides `/verify`, `/settle`, and `/supported` |
| Registry client | `worker/src/solana/registry.ts` | Reads agent endpoint, pricing, split, protocol, and payout metadata |
| Revenue math | `worker/src/revenue.ts` | Computes owner, buyback, treasury, and operator split amounts |
| Holder discounts | `worker/src/clawd.ts` | Applies `$CLAWD` balance discounts with payer binding |
| A2A bridge | `worker/src/protocols/a2a.ts` | Payment-gates Google-style A2A JSON-RPC calls |
| MPP bridge | `worker/src/protocols/mpp.ts` | Maps `WWW-Authenticate: Payment` flows to Solana exact payments |
| AP2 bridge | `worker/src/protocols/ap2.ts` | Verifies AP2 intent mandates before Solana settlement |
| Client SDK | `sdk/src/index.ts` | `clawdFetch` client that detects 402s, signs, and retries |
| Vault program | `programs/clawd-vault/src/lib.rs` | Anchor registry and revenue vault |

## Security Invariants

- Use the same discounted amount in challenge and settlement verification.
- Bind discounts to the actual payer wallet.
- Reject non-HTTPS upstream endpoints and localhost/private IPv4 targets.
- Reject malformed registry accounts and invalid pricing.
- Validate resource, amount, decimals, recipient, asset, and p-token batch outputs before signing.
- Strip hop-by-hop headers and payment credentials before proxying upstream.
- Persist receipts without leaking private keys or signed transaction material beyond what is needed for audit.
- Require all split basis points to total `10000`.

## Default Revenue Split

| Recipient | Default |
|---|---:|
| Agent owner | 70% |
| `$CLAWD` buyback | 15% |
| Treasury | 10% |
| Operator | 5% |

## Environment

| Variable | Purpose |
|---|---|
| `NETWORK` | `solana-mainnet` or `solana-devnet` |
| `TREASURY_OWNER` | Recipient owner for gateway challenge payments |
| `USDC_MINT` | SPL USDC mint for default pricing |
| `CLAWD_MINT` | `$CLAWD` mint for holder discounts |
| `CLAWD_VAULT_PROGRAM` | Anchor registry/vault program id |
| `P_TOKEN_PROGRAM_ID` | Optional p-token program override |
| `USE_P_TOKEN` | Disable to force SPL Token transfers |
| `REGISTRY_SEED` | PDA seed for registry reads |
| `PINATA_GATEWAY` | Gateway for manifest and receipt JSON |
| `SPLIT_*_BPS` | Split basis points; must total `10000` |
| `DISCOUNT_TIER_*` | Holder thresholds and discount bps |
| `HELIUS_API_KEY` | Primary Solana RPC provider |
| `SOLANATRACKER_API_KEY` | RPC fallback provider |
| `PINATA_JWT` | Pins settlement receipts |
| `AP2_VERIFIER_JWK` | Public JWK for AP2 mandate verification |

## Endpoints

| Route | Purpose |
|---|---|
| `GET /health` | Liveness and network metadata |
| `GET /registry/:id` | Reads registered agent account |
| `POST /facilitator/verify` | Verifies payment against a challenge |
| `POST /facilitator/settle` | Re-verifies and broadcasts the payment transaction |
| `GET /facilitator/supported` | Lists supported networks, assets, and token programs |
| `ANY /agents/:id/*` | Payment-gated proxy to registered upstream agent |
| `GET /a2a/:id/.well-known/agent.json` | Free A2A card lookup |
| `POST /a2a/:id` | Payment-gated A2A JSON-RPC call |
