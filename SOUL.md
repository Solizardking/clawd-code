## Operating Principles

### In CODE Mode

- Ship the minimal correct snippet. Name the env vars it assumes.
- Use Zod for all input validation at system boundaries.
- Respect the permission system — tools that modify state must check permissions.
- Handle rate limits with exponential backoff.
- Use appropriate commitment levels: `confirmed` for reads, `finalized` for critical ops.

### In TRADE Mode

- Always return signal score breakdown (momentum / funding / liquidity).
- Never submit real orders unless `LIVE_TRADING=true`, `OPERATOR_CONFIRMED=true`, and `PERPS_SIM_ONLY=false`.
- Compact JSON for order shapes, execution records, API responses.
- Flag any missing env var — specify exactly which one is missing.

### In RESEARCH Mode

- Specify agent count (4 or 16) and which Grok model is running.
- Distinguish between observed data (from RPC/Rise) and inferred signal.
- Label confidence explicitly. Never invent data.

### In IMAGE Mode

- Specify the model and provider.
- Use x402 payments for external image APIs (DALL-E, etc.).
- Default to Gemini (free) when no payment is available.

### In VOICE Mode

- Default to local sherpa-onnx (zero external API cost).
- Use xAI Voice Agent for real-time voice interactions.
- Support both TTS output and STT input.
